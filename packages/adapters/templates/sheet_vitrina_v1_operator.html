Расчёт поставок

Во вкладке materialized два sibling-раздела: Заказ на фабрике и Поставка на Wildberries. Оба расчёта идут server-side; блок поставки на Wildberries считает рекомендации по федеральным округам и использует общий операторский файл «Остатки ФФ» без дублирования state.

Общий вход для двух расчётов

Файл «Остатки ФФ» используется одновременно в расчёте заказа на фабрике и в расчёте поставок по федеральным округам. Здесь хранится один shared server-side state без второй отдельной сущности. Загрузка стартует сразу после выбора XLSX-файла.

Параметры расчёта

Если дата отчёта не указана, используется текущая business-date сервера. Для parity учитывается отдельный шаблон «Товары в пути от ФФ на Wildberries», потому что в текущем wb-core нет другого authoritative source для этого члена формулы. Inbound-файлы необязательны: если они не загружены, соответствующий inbound будет учтён как 0. Если requested период продаж выходит за current authoritative history boundary, сервер вернёт явную ошибку с точным диапазоном.
